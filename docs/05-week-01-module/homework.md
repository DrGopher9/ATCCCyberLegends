# Week 1 — Homework (solo, before the weekend lab / Week 2)

Do this alone. The point is to prove to yourself you can do Week 1's skills *without* someone next to
you — because in competition, nobody will be. Budget ~1 hour.

## 1. Repo workflow, from memory (~20 min)
Without looking at [`lab-exercises.md`](lab-exercises.md) unless you're stuck:
- [ ] `git pull origin main`
- [ ] Create a branch: `git checkout -b week1hw-<yourname>`
- [ ] Add one line to your scratch file: something you learned this week
- [ ] Commit with a clear message and push **your branch** (not `main`)

If you had to peek, do it once more without peeking.

## 2. Lab reachability (~15 min)
- [ ] Connect to the lab unaided
- [ ] Pick any one Linux box and answer: `whoami`, `hostname`, `ip a`, `ss -tlnp`
- [ ] Write down: which box, its IP, and one service it's running

## 3. Rules of engagement (~15 min)
Without notes, write down:
- [ ] One sentence: what is CCDC and how is it scored?
- [ ] **Three** actions that cause point loss or disqualification
- [ ] One thing you're *allowed* to do (from the ✅ list)

## 4. Look ahead (~10 min)
- [ ] Skim [`../04-curriculum/phase-0-foundations.md`](../04-curriculum/phase-0-foundations.md) so you
      know what Week 2 (Linux fundamentals) will cover.
- [ ] Note one thing you're unsure about to ask at the next session.

## Bring to the weekend lab / Week 2
- Your three penalties written down (needed for the assessment)
- Any question that came up
- The name of the lab box you inspected

> Stuck on anything? That's expected in Week 1 — message the team channel or your paired partner.
> Getting unstuck early is the whole habit we're building.
