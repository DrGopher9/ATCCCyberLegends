# Week 1 — Assessment

Run during the weekend lab. The member **performs** each item while a coach or a peer observes — no
describing, no notes for the hands-on parts. Mark ✅ pass or 🔁 needs-another-rep. Record results in
the team rubric-tracking sheet.

**Covers rubric items:** `T1-G1`, `T1-G2` ([`../02-readiness-rubric.md`](../02-readiness-rubric.md)).

---

## Part A — `T1-G1`: Reach the lab & use git (hands-on, unaided)

Time budget: **5 minutes.** The member, without help:

| # | Task | ✅/🔁 |
|---|---|---|
| A1 | `git pull origin main` in the repo | |
| A2 | Create a branch named for themselves | |
| A3 | Make a change and commit it with a clear message | |
| A4 | Push **their branch** (and does *not* push to `main`) | |
| A5 | Connect to the lab and run `hostname` + `ss -tlnp` on a box | |

**Pass A** = all five ✅ within the time budget.

## Part B — `T1-G2`: Rules of engagement (verbal/written, from memory)

| # | Task | ✅/🔁 |
|---|---|---|
| B1 | State in one sentence what CCDC is and how it's scored | |
| B2 | Name **three** actions that cause point loss or disqualification | |
| B3 | Name one thing the team **is** allowed to do | |

**Pass B** = B1 correct + B2 lists three valid items + B3 correct.

---

## Scoring

- **Both A and B pass →** `T1-G1` and `T1-G2` marked ✅. Member is on track for the Tier-1 gate (Week 7).
- **One or both 🔁 →** note exactly which items, pair the member with a partner, and re-check at the
  start of Week 2. No shame — Week 1 is orientation; the point is to *close the gap early*, not to
  rank people.
