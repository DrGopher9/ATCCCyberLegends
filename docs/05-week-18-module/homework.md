# Week 18 — Homework (solo, before the weekend lab / Week 19)

Budget ~1 hour. Documentation is scored and it's what makes your mistakes recoverable. Next week is the
mini-scrimmage — docs will be under real pressure.

## 1. Change-log discipline (~20 min)
- [ ] Make three real changes on your box, logging each as you go with enough detail to undo it
- [ ] Then roll one back **using only your log entry**; verify the service still works
- [ ] If you couldn't undo it from the log, rewrite the entry so you could

## 2. Write a scorable incident report (~25 min)
For a real or replayed incident on your box:
- [ ] Capture source/dest IP + timeline first
- [ ] Write the report: src/dst IP, timeline, creds cracked, impact, remediation
- [ ] Keep it ~1 page, exploitation-focused, **not padded**; save as PDF
- [ ] Self-check: would a judge call it clear, thorough, accurate — and not frivolous?

## 3. Handoff practice (~15 min)
- [ ] Write a handoff note for your box as if you're going on break: what changed, what's broken, where
      the log is
- [ ] Ask your backup if they could pick up from it cold

## Bring to the weekend lab / Week 19
- Your IR report (PDF)
- Your handoff note
- Any change you couldn't roll back from your log (and how you fixed the entry)
